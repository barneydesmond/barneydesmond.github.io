Sending mail via Anchor's servers

In general we recommend using the outgoing mail server provided by your ISP, since it generally requires less configuration and works without issue.

There are some cases, however, where this doesn't work out so well. These usually are because:

  • Some ISPs don't offer an outgoing mail server
  • The ISP's outgoing mail server is sub-standard
  • The device used for mail frequently moves between networks

For these cases you can, with a little extra configuration, use Anchor as your outgoing mail server.

Settings to use

Setting name Value
Hostname smtp.anchor.net.au
Port 587
Encryption TLS
Username/Password Same as your incoming username (ie. your email address/password)

Port 587 is the standard for mail submission, but it may be blocked from some networks. In those cases, there alternatives available:

  • Port 25 (with TLS encryption)
  • Port 465 (with SSL encryption)
